# UX/UI Standards

This product should feel like a high-trust enterprise HR system: calm, precise, confidential, and fast to scan.

## Typography

- Use `Inter` for Latin text and `Noto Sans Thai` for Thai text.
- Prefer readable body text over compressed UI density.
- Avoid negative letter spacing.
- Use uppercase only for short labels, badges, and metadata.

## Layout

- Primary pages use a constrained content width with a stable shell.
- Operational pages should lead with the working surface, not marketing content.
- Tables should preserve scanability: clear status, score, owner, and primary action.
- Repeated actions should be grouped into menus when more than two export/action choices exist.

## Governance Signals

- Every protected page shows confidentiality, compliance, and audit context.
- Export surfaces must communicate document type clearly.
- Salary and bonus data must stay role-gated.

## Accessibility

- Keyboard users must be able to skip to main content.
- Buttons need clear labels and stable dimensions.
- Text should not rely on color alone to communicate state.
- Focus rings must remain visible.

## Export Experience

- CSV is the structured data artifact.
- PDF is the executive/report artifact.
- PDF exports should offer language variants where required.
- PDF layout must include title, score summary, sections, page numbers, and confidentiality framing.
